Каждая подложка несёт случайную каплю вещества — «кляксу» с уникальным рисунком интерференционных полос. Система запоминает этот рисунок и по одному новому снимку определяет: какая это подложка и что на ней записано в базе данных.

```
whattw query фото_подложки.jpg

  Rank  Blob ID                   Inliers    Rotation
  ────────────────────────────────────────────────────
  1     sample_42                    4764          0°  ◀
  2     sample_07                      18        +90°
  3     sample_15                       9          0°

[✓] MATCH → 'sample_42'  (4764 совпадений)
```

---

## Как это работает

Система не обучается заново при добавлении новых подложек. Вместо этого она использует две предобученные модели компьютерного зрения:

- **LoFTR** — находит плотную сетку совпадающих точек между двумя снимками
- **DISK + LightGlue** — находит ключевые точки и сопоставляет их точечно

При добавлении подложки (`add`) — модели один раз извлекают «цифровой отпечаток» снимка и сохраняют его в базе. При запросе (`query`) — отпечаток нового снимка сравнивается с базой, побеждает тот, у кого больше геометрически согласованных совпадений.

Система автоматически проверяет все повороты на 90° — если подложку сфотографировали иначе, чем добавляли, это не проблема.

## Установка

**Требования:** Python 3.10+, pip

```bash
cd WhatTheWafer

# Установить зависимости и зарегистрировать команду whattw
```

## Быстрый старт

### Шаг 1 — Добавить эталонные снимки

Сфотографируйте каждую подложку 2–4 раза (с разных ракурсов, при разном освещении) и добавьте в базу:

```bash
whattw add фото1.tif --new sample_42
whattw add фото2.tif --wafer sample_42   # второй снимок той же подложки
whattw add другая.tif --new sample_07
```

Принимаются форматы: **TIF, JPEG, PNG, BMP**.

### Шаг 2 — Идентифицировать неизвестный снимок

```bash
whattw query неизвестная.jpg
```

### Шаг 3 — Визуально проверить совпадение

```bash
whattw compare эталон.tif неизвестная.jpg --output result.png
```

Команда сохраняет трёхпанельное изображение: совпадения DISK+LightGlue, совпадения LoFTR и вердикт.

## Все команды

### `add` — добавить подложку в базу

```bash
whattw add <снимок> --new <имя>              # создать новую подложку
whattw add <снимок> --wafer <имя|uuid>       # добавить снимок к существующей
```

| Флаг | Описание |
|---|---|
| `--new ИМЯ` | Создать новую подложку с этим идентификатором, например `sample_42` |
| `--wafer ИМЯ\|UUID` | Имя или 4-символьный UUID существующей подложки |
| `--force` | Добавить без предупреждения о дубликате |
| `--matcher disk\|loftr\|both` | Матчер для проверки дубликатов (по умолч.: `disk`) |

### `query` — идентифицировать снимок

```bash
whattw query <снимок>
```

| Флаг | Описание |
|---|---|
| `--top-k N` | Показать N лучших совпадений (по умолч.: 5) |
| `--threshold N` | Минимальное число совпадений для положительного ответа (по умолч.: 15) |
| `--matcher disk\|loftr\|both` | Какой матчер использовать (по умолч.: `both`) |
| `--fine-rotation` | Дополнительный поиск в диапазоне ±5/10/15° вокруг лучшего поворота |
| `--debug` | Показать подробные счётчики по каждому эталону |

### `compare` — визуальное сравнение двух снимков

```bash
whattw compare <снимок1> <снимок2> --output result.png
```

| Флаг | Описание |
|---|---|
| `--output`, `-o` | Путь к выходному PNG (по умолч.: `compare_result.png`) |
| `--fine-rotation` | Точный поиск поворота |

### `list` — список подложек в базе

```bash
whattw list
```

### `clear` — удалить из базы

```bash
whattw clear --wafer sample_42            # удалить одну подложку (по имени или UUID)
whattw clear --wafer sample_42 --image 1  # удалить один снимок (индекс из list)
whattw clear --all                        # очистить всю базу
```

### Глобальный флаг `--no-gpu`

```bash
whattw --no-gpu query неизвестная.jpg
```

Принудительно использует CPU, даже если GPU доступен.

## Советы

| Ситуация | Рекомендация |
|---|---|
| Съёмка в тёмном поле | Добавьте хотя бы один тёмнопольный снимок той же подложки |
| Очень крупный кадр | Добавьте крупноплановый снимок как дополнительный эталон |
| Снимок немного скошен (< 15°) | Используйте `--fine-rotation` |
| Подложка не находится | Запустите с `--debug` — будут видны счётчики по каждому эталону |
| Ложное срабатывание при `add` | Используйте `--force` для принудительного добавления |

**Сколько эталонных снимков нужно?**  
Минимум 1, оптимально 2–4. Добавляйте снимки при разных условиях съёмки — освещение, масштаб, ориентация. Чем разнообразнее эталоны, тем выше устойчивость к вариациям.

## Структура проекта

```
WhatTheWafer/
├── wafer_id.py          — точка входа, CLI
├── preprocessing.py     — загрузка и сегментация изображений

├── visualizer.py        — визуализация совпадений
├── matchers/
│   ├── loftr_matcher.py          — плотный матчер LoFTR
│   └── disk_lightglue_matcher.py — точечный матчер DISK+LightGlue
├── models/
│   └── checkpoints/     — веса моделей (~95 MB, поставляются с проектом)
├── database/            — база данных подложек (создаётся автоматически)
├── Dockerfile           — для сборки переносимого образа
└── DEPLOY.md            — инструкция переноса на изолированный ПК
```

## Технические детали

| | |
|---|---|
| Модели | LoFTR (indoor), DISK (depth), LightGlue |
| Хранилище | SQLite (метаданные) + HDF5 (признаки) + FAISS FlatL2 (быстрый поиск) |
| Форматы входных данных | TIF (16-bit, OME), JPEG, PNG, BMP |
| Скорость query | ~0.5 с на GPU при любом размере базы (FAISS отсеивает кандидатов) |
| Переобучение при `add` | Нет — модели заморожены, `add` только извлекает признаки |
