Каждая подложка несёт на себе особенность: клякса от маркера в углу с уникальным рисунком интерференционных полос. Система запоминает этот «цифровой отпечаток» и записывает в базу данных. По единственному новому снимку определяет: какая это подложка и что о ней записано.

```
whattw query фото_подложки.jpg

  Rank  Blob ID                   Inliers    Rotation
  ──────────────────────────────────────────────────
  1     sample_42                      87          0°  ◀
  2     sample_07                       6        +90°
  3     sample_15                       3          0°

[✓] MATCH → 'sample_42'  (87 инлайеров)
```

---

## Как это работает

Система не обучается заново при добавлении новых подложек. Используются две замороженные предобученные модели:

- **DISK** — находит до 2048 характерных ключевых точек на снимке, отранжированных по уверенности
- **LightGlue** — сопоставляет ключевые точки между двумя снимками с помощью графовой нейросети

При добавлении подложки (`add`) — модели один раз извлекают отпечаток и сохраняют. При запросе (`query`) — индекс FAISS сужает базу до 5 наиболее похожих кандидатов, затем LightGlue оценивает каждого с RANSAC-верификацией геометрии. Побеждает тот, у кого больше инлайеров.

Система автоматически перебирает все 4 поворота по 90° — если подложку сфотографировали иначе, чем при добавлении, это не проблема.

## Быстрый старт

### Шаг 1 — Добавить эталонные снимки

Сфотографируйте каждую подложку 2–4 раза (с разных ракурсов, при разном освещении) и добавьте в базу.  
Можно передать несколько снимков сразу — индекс перестроится только один раз:

```bash
# Создать новую подложку с несколькими снимками за один раз
whattw add фото1.tif фото2.tif фото3.tif --new sample_42

# Или добавлять по одному
whattw add другая.tif --new sample_07
whattw add другая2.tif --wafer sample_07   # второй снимок той же подложки
```

Принимаются форматы: **TIF (16-bit, OME), JPEG, PNG, BMP**.

### Шаг 2 — Идентифицировать неизвестный снимок

```bash
whattw query неизвестная.jpg
```

### Шаг 3 — Визуально проверить совпадение

```bash
whattw compare эталон.tif неизвестная.jpg --output result.png
```

Сохраняет изображение «бок о бок»: все совпадения LightGlue (серые) и RANSAC-инлайеры (зелёные).

## Все команды

### `add` — добавить подложку в базу

```bash
whattw add <снимок> [<снимок> ...] --new <имя>           # создать новую подложку
whattw add <снимок> [<снимок> ...] --wafer <имя|uuid>    # добавить снимки к существующей
```

| Флаг | Описание |
|---|---|
| `--new ИМЯ` | Создать новую подложку с этим идентификатором, например `sample_42` |
| `--wafer ИМЯ\|UUID` | Имя или 4-символьный UUID существующей подложки |
| `--force` | Добавить без предупреждения о дубликате |

Можно передать несколько путей к файлам — FAISS перестраивается один раз в конце.

### `query` — идентифицировать снимок

```bash
whattw query <снимок>
```

| Флаг | Описание |
|---|---|
| `--top-k N` | Показать N лучших совпадений (по умолч.: 5) |
| `--threshold N` | Минимальное число инлайеров для положительного ответа (по умолч.: 15) |
| `--fine-rotation` | Дополнительный поиск в диапазоне ±5/10/15° вокруг лучшего поворота |
| `--debug` | Показать подробные счётчики по каждому эталону |

### `compare` — визуальное сравнение двух снимков

```bash
whattw compare <снимок1> <снимок2> --output result.png
```

| Флаг | Описание |
|---|---|
| `--output`, `-o` | Путь к выходному PNG (по умолч.: `compare_result.png`) |
| `--fine-rotation` | Точный поиск поворота вокруг лучшего кардинального угла |

### `list` — список подложек в базе

```bash
whattw list
```

Выводит каждую подложку с коротким UUID, количеством видов, ключевых точек и заметкой (если есть).

### `memory` — добавить или обновить заметку для подложки

```bash
whattw memory <имя|uuid|blob_id> "короткая заметка об этой подложке"
whattw memory sample_42 ""   # очистить заметку
```

Заметка отображается в `list` и в попапе демона при успешной идентификации.
Принимает имя подложки, 4-символьный короткий UUID или полный blob ID.

### `daemon` — фоновый демон с хоткеем

```bash
whattw daemon
whattw daemon --threshold 60
```

Запускает фоновый процесс, который слушает комбинацию **Ctrl+Shift+F9**.

**Порядок работы:**
1. Сделайте скриншот системным инструментом (PrtSc / выделение области на Linux; PrtSc / Win+Shift+S на Windows)
2. Нажмите **Ctrl+Shift+F9** — демон читает буфер обмена, идентифицирует подложку и показывает попап в правом верхнем углу:
   - Обрезанный снимок из запроса (автоматически повёрнут, чтобы совпадать с ориентацией в базе)
   - Миниатюра из базы данных рядом
   - Заметка под снимками (если есть)

Попап остаётся на экране до нажатия **Escape**, клика мышью или повторного нажатия **Ctrl+Shift+F9** (которое сразу начинает новую идентификацию).

Модели загружаются один раз при запуске (~10 с); все последующие идентификации — мгновенные.

| Флаг | Описание |
|---|---|
| `--threshold N` | Минимальное число инлайеров для уверенного совпадения (по умолч.: 50) |
| `--no-gpu` | Принудительно использовать CPU |
| `--debug` | Сохранить промежуточные обрезки и маски во временную папку системы |

**Linux:** демон читает `/dev/input` напрямую через `evdev` (работает на Wayland и X11). Если видите ошибку прав доступа:
```bash
# Вариант А — только для текущей сессии:
sudo whattw daemon

# Вариант Б — постоянно (требует повторного входа):
sudo usermod -aG input $USER
```

**Windows:** используется `pynput` — дополнительная настройка не нужна.

### `clear` — удалить из базы

```bash
whattw clear --wafer sample_42            # удалить одну подложку (по имени или UUID)
whattw clear --wafer sample_42 --image 1  # удалить один снимок (индекс из list)
whattw clear --all                        # очистить всю базу
```

| Флаг | Описание |
|---|---|
| `--wafer ИМЯ\|UUID` | Подложка для удаления |
| `--image N` | Индекс снимка для удаления (используется вместе с `--wafer`) |
| `--all` | Очистить всю базу |
| `--yes`, `-y` | Пропустить подтверждение |

### Глобальный флаг `--no-gpu`

```bash
whattw --no-gpu query неизвестная.jpg
```

Принудительно использует CPU, даже если GPU доступен.

## Советы

| Ситуация | Рекомендация |
|---|---|
| Съёмка в тёмном поле | Добавьте хотя бы один тёмнопольный снимок той же подложки как эталон |
| Очень крупный кадр | Добавьте крупноплановый снимок как дополнительный вид |
| Снимок немного скошен (< 15°) | Используйте `--fine-rotation` |
| Подложка не находится | Запустите с `--debug` — будут видны счётчики по каждому эталону |
| Ложное срабатывание при `add` | Используйте `--force` для принудительного добавления |

**Сколько эталонных снимков нужно?**  
Минимум 1, оптимально 2–4. Добавляйте снимки при разных условиях съёмки — освещение, масштаб, ориентация. Чем разнообразнее эталоны, тем выше устойчивость к вариациям.

## Как устроены модели

### DISK — детектор и дескриптор ключевых точек

DISK (Deep Image local featureS with Keypoints) — свёрточная нейронная сеть, обученная сквозным образом на одновременное обнаружение характерных точек и вычисление их дескрипторов. В отличие от классических детекторов вроде SIFT или ORB — которые используют вручную написанные формулы на основе градиентов и эвристики — DISK обучается напрямую максимизировать количество геометрически корректных совпадений между парами снимков.

Архитектура построена по принципу U-Net: энкодер сжимает снимок в компактную карту признаков, декодер восстанавливает разрешение до исходного, возвращая для каждого пикселя вероятность детекции и 128-мерный вектор дескриптора. Во время обучения используются пары снимков с известной относительной геометрией. Ключевые точки, которые успешно сопоставляются, получают награду; точки, которые не находят пару, — штраф. Этот сигнал обратно распространяется через дифференцируемое приближение RANSAC, поэтому сеть учится не просто «куда смотреть», но «какие точки выживут при геометрической верификации».

Предобученные веса `depth`, используемые здесь, обучены на датасете MegaDepth — большой коллекции 3D-реконструкций сцен из фотографий из интернета. Несмотря на обучение на естественных сценах, признаки хорошо переносятся на микроскопические снимки: рисунки интерференционных полос создают богатую высококонтрастную локальную текстуру, которую сеть уверенно обнаруживает и описывает.

На инференсе DISK возвращает до 2048 ключевых точек на снимок, отранжированных по уверенности детектора. Хранятся только лучшие — это хороший баланс между скоростью и покрытием.

### LightGlue — матчер ключевых точек

LightGlue — графовая нейронная сеть, которая принимает два набора ключевых точек с их дескрипторами и возвращает разреженное множество соответствий. Разработана как лёгкая замена SuperGlue с акцентом на скорость без потери точности.

Сопоставление происходит итеративно через стек трансформерных слоёв. Каждый слой состоит из двух фаз:
- **Само-внимание (self-attention)** — каждая точка обращает внимание на все остальные точки *того же* снимка, накапливая геометрический контекст (например, сеть учится, что определённые пространственные конфигурации — надёжные якоря).
- **Перекрёстное внимание (cross-attention)** — каждая точка обращает внимание на все точки *другого* снимка, постепенно формируя уверенность о том, с какой точкой она должна быть сопоставлена.

Ключевая особенность LightGlue — **адаптивная глубина**: сеть может прекратить обработку отдельных пар точек как только уверена в их сопоставлении, не прогоняя все слои для всех пар. Это существенно ускоряет инференс при большом количестве очевидных совпадений — что типично для повторных снимков одной подложки.

LightGlue специально обучен работать с дескрипторами DISK (а также SuperPoint), то есть обе модели образуют согласованную пару, оптимизированную совместно. На выходе — список пар индексов совпавших точек, которые затем передаются в RANSAC для фильтрации геометрической согласованности.

### Зачем RANSAC поверх LightGlue?

LightGlue уже учитывает геометрию через механизм внимания, но всё же может вернуть небольшое количество ошибочных соответствий. RANSAC (Random Sample Consensus) подбирает модель гомографии к предложенным совпадениям и классифицирует каждое как инлайер или аутлайер на основе ошибки перепроецирования. Количество инлайеров и есть финальный счёт, по которому происходит идентификация. Двухэтапный подход (нейросетевое сопоставление → классическая геометрическая верификация) устойчив к изменениям освещения, частичному перекрытию и оптическим искажениям, характерным для микроскопии.

### Требуется ли дообучение?

**Нет.** И DISK, и LightGlue используются в полностью замороженном виде — никакого дополнительного обучения при добавлении новых подложек не происходит.

Это работает по нескольким причинам:
- **DISK** обучен на большом разнообразном датасете и извлекает универсальные признаки, хорошо переносимые на новые задачи. Рисунки интерференционных полос создают характерную локальную структуру, которую DISK надёжно находит без какой-либо адаптации.
- **LightGlue** обучен на парах снимков с известными геометрическими преобразованиями и учится рассуждать о геометрической согласованности — что напрямую применимо к задаче идентификации подложек.
- **RANSAC** — жёсткий геометрический фильтр, полностью не зависящий от предметной области.

Единственный специфичный для задачи компонент — предобработка (сегментация кляксы, обрезка, масштабирование). Это классическое компьютерное зрение, не требующее обучения. Добавление новой подложки — это исключительно прямой проход через сеть: извлечь признаки один раз, сохранить. Никакого вычисления градиентов, никакого обновления весов, никакого цикла обучения.

Дообучение DISK или LightGlue на снимках подложек теоретически могло бы добавить несколько инлайеров в сложных случаях. Но это потребовало бы подготовленных парных обучающих данных (одна подложка, несколько условий съёмки, с известной относительной геометрией) — что дорого и трудоёмко. При этом замороженные модели уже обеспечивают устойчивую идентификацию на практике.

## Технические детали

| | |
|---|---|
| Модели | DISK (depth), LightGlue |
| Ключевые точки на снимок | до 2048 (лучшие по уверенности детектора) |
| Хранилище | SQLite (метаданные) + HDF5 float16 (признаки) + FAISS ScalarQuantizer QT_8bit (быстрый поиск) |
| Размер БД | ~0.8 MB на снимок |
| Форматы входных данных | TIF (16-bit, OME), JPEG, PNG, BMP |
| Скорость query | Секунды на GPU при любом размере базы (FAISS ограничивает кандидатов до 5) |
| Переобучение при `add` | Нет — модели заморожены, `add` только извлекает признаки |
